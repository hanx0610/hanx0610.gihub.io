BIM产品前端开发规范与注意事项

发表于 2018-09-17 | 分类于 前端

版本 1.0

本规范仅适用于BIM业务线内Vue开发体系下的WEB产品
浏览器分辨率最低支持为1280 x 720
浏览器最低版本为IE11
开发采用优雅降级策略
使用ES6作为JS语法标准
编码UTF-8格式
编码风格遵守eslint

Html 规范

不允许使用任何形式的外链
通常情况下不允许使用iframe
减少DOM元素数量
本地存储用来存储大量的数据，比如本地邮箱
本地存储不允许存放敏感数据

Css 规范

当一个样式规则包含多个 selector 时，每个选择器必须独占一行
属性定义必须另起一行
使用4个空格作为一个缩进层级
建议外层框架结构使用position布局
不滥用Float（时计算量比较大）
不滥用@import(影响CSS加载速度)
不滥用高级选择器(计算量大)
不滥用Web字体（会重新绘制当前界面）
利用CSS继承特性
值为0时不需要任何单位
开启GPU加速。（css的transitions和transforms）
缩进样式从而反映 DOM；
使用Sass时应使用Sass语法格式
使用Sass时应对变量进行统一管理
Debugging：如果你要解决 CSS 问题的话，先把旧代码拿掉再写新的

JavaScript 规范

使用ES6以下语法
任何时候都需要明确作用域,使用局部变量
声明变量必须加上 var 关键字.
变量声明尽量提在函数首部，不允许出现连着的两个var声明(也包括let，const)
请使用字面量值创建对象
使用拓展运算符 … 复制数组
常量必须大写并用 const 声明
没必要在每次声明变量时就将其初始化
不要使用with
不要使用eval()
不要使用IE下的条件注释
不要直接使用undefined进行变量判断，使用typeof和字符串’undefined’对变量进行判断
不要修改内置对象, 如 Object.prototype 和 Array.prototype 的原型
使用 Array 和 Object 语法, 而不使用 Array 和 Object 构造器.
对resize和scroll事件使用函数节流
使用匿名回调函数（函数执行完内存回收）
dom 要批量插入，修改
不要滥用闭包，手工解除变量引用
减少Cookie的大小
只有特殊的异常是需要处理的
需要提交的数据需要进行格式与非空验证
可以使用三元操作符
二元布尔操作符是可短路的, 只有在必要时才会计算到最后一项.
数组和对象的初始化如果初始值不是很长, 就保持写在单行上:
初始值占用多行时, 缩进2个空格.
使用单引号：单引号优于双引号. 当你创建一个包含 HTML 代码的字符串时就知道它的好处了.
调试信息console.log() debugger 使用完及时删除

Vue 规范

使用指令缩写 (用 : 表示 v-bind: 和用 @ 表示 v-on:)
使用 export ，import 模块化
使用 type 属性校验类型
使用 props 之前先检查该 prop 是否存在
避免 this.$parent
避免 v-if 和 v-for 用在一起
避免使用JSX方式生成页面,使用VUE推荐的方式
谨慎使用 this.$refs
组件样式设置作用域一个基于 class 的类似 BEM 的策略
组件的 data 必须是一个函数。
不得随意安装第三方组件
为v-for设置键值

关于 vuex 的 ajax

首先 ajax 请求可以写在 actions也可以直接写在 .vue 页面里;
判断的依据是 回调是否需要调用页面结构来区分,
比如在 .vue 页面中 发送完请求后需要调用 this.$refs.element等,或者需要利用组件的独立性的效果时 的那就写在.vue页面,否则就写在 actions 里

模板中简单的表达式

组件模板应该只包含简单的表达式，复杂的表达式则应该重构为计算属性或方法。

带引号的特性值

非空 HTML 特性值应该始终带引号 (单引号或双引号，选你 JS 里不用的那个)。
在 HTML 中不带空格的特性值是可以没有引号的，但这样做常常导致带空格的特征值被回避，导致其可读性变差。

关于mounted、created

不在 mounted、created 之类的方法写逻辑，取 ajax 数据，
在 created 里面监听 Bus 事件
Created：数据的初始化，异步请求调用
Mounted: 通常是初始化页面完成后，再对dom节点进行一些需要的操作
methods里面定义的函数，是需要主动调用的
watch擅长一个数据影响多个数据
computed擅长一个数据受多个数据影响
methods不处理数据逻辑关系，只提供可调用函数

通讯与跳转

携参跳转只允许使用path通常只携带id等简单非敏感数据,
无参跳转用name
父与子通信：使用props
子与父通信：父组件监听机制
非父子组件通信: 使用Vue-Bus
复杂通讯使用：VueX

data props 方法注意点

使用 data 里的变量时请先在 data 里面初始化
props 指定类型，也就是 type
props 改变父组件数据 基础类型用 $emit ，复杂类型直接改
ajax 请求数据用上 isLoading、isError 变量
不命名多余数据，现在是详情页、你的数据是 ajax 请求的，那就直接声明一个对象叫 d，而不是每个字段都声明
表单数据请包裹一层 form

Vue方法放置顺序

components

data

mounted
activited
update
beforeRouteUpdate
metods
filter

watch

vuex 的dispatch和commit提交mutation的区别

一个异步操作与同步操作的区别。
当操作行为中含有异步操作，比如向后台发送请求获取数据，就需要使用action的 dispatch去完成了。
其他使用commit即可。
commit=>mutations,用来触发同步操作的方法。
dispatch =>actions,用来触发异步操作的方法。

命名规范

单文件组件的文件名应该是单词大写开头

模板中的组件名大小写总是 PascalCase 的

组件名应该倾向于完整单词而不是缩写

组件名应该始终是多个单词的，根组件 App 除外。

基础组件名

应用特定样式和约定的基础组件 (也就是展示类的、无逻辑的或无状态的组件) 应该全部以一个特定的前缀开头，比如 Base、App 或 V。

views 下的文件夹命名

views 下面的文件夹代表着模块的名字
由名词组成（car、order、cart）
单词只能有一个（good: car order cart）（bad: carInfo carpage）
尽量是名词（good: car）（bad: greet good）
以小写开头（good: car）（bad: Car）

views 下的 vue 文件命名

views 下面的 vue 文件代表着页面的名字
放在模块文件夹之下
只有一个文件的情况下不会出现文件夹，而是直接放在 views 目录下面，如 Login Home
尽量是名词
大写开头，开头的单词就是所属模块名字（CarDetail、CarEdit、CarList）
名字至少两个单词（good: CarDetail）（bad: Car）
常用结尾单词有（Detail、Edit、List、Info、Report）
以 Item 结尾的代表着组件（CarListItem、CarInfoItem）

Vue Method

动宾短语（good：jumpPage、openCarInfoDialog）（bad：go、nextPage、show、open、login）
ajax 方法以 get、post 开头，以 data 结尾（good：getListData、postFormData）（bad：takeData、confirmData、getList、postForm）
事件方法以 on 开头（onTypeChange、onUsernameInput）
init、refresh 单词除外
尽量使用常用单词开头（set、get、open、close、jump）
驼峰命名（good: getListData）（bad: get_list_data、getlistData）

Css命名

使用连字符(-)连接class名字。例如：(.foo-bar 而非 .foo_bar)，其中第一组单词应为抽象的元素，第二组单词应为不同的状态或事物动作。例如：（.page_left，room-red）
缩进：采用4个空格
命名避免使用中文拼音，应该采用更简明有语义的英文单词进行组合
命名注意缩写，但是不能盲目缩写，具体请参见常用的CSS命名规则
不允许通过1、2、3等序号进行命名
避免class与id重名
id用于标识模块或页面的某一个父容器区域，名称必须唯一，不要随意新建id
class用于标识某一个类型的对象，命名必须言简意赅。
尽可能提高代码模块的复用，样式尽量用组合的方式
规则名称中不应该包含颜色（red/blue）、定位（left/right）等与具体显示效果相关的信息。应该用意义命名，而不是样式显示结果命名。
代码顺序：尽量按照特定顺序编写规则，这将确保你充分发挥 CSS 缩写中第一个 C 的意义：cascade，层叠。有前缀的声明适当缩进，从而对齐其值；

JavaScript 变量

必须采用小驼峰式命名法。
命名规范：前缀应当是名词。(函数的名字前缀为动词，以此区分变量和函数)
命名建议：尽量在变量名字中体现所属类型，如:length、count等表示数字类型；而包含name、title表示为字符串类型。

JavaScript 函数

必须采用小驼峰式命名法。
命名规范：前缀应当为动词。
命名建议：可使用常见动词约定

JavaScript 私有函数与属性

私有属性和方法：前缀为_(下划线)，后面跟公共属性和方法一样的命名方式。

命名：Getters 和 setters 并不是必要的. 但只要使用它们了, 就请将 getters 命名成 getFoo() 形式,
将 setters 命名成 setFoo(value) 形式. (对于布尔类型的 getters, 使用 isFoo() 也可.)

JavaScript常量

必须采用全大写的命名，且单词以_分割

类 & 构造函数

大驼峰式命名法，首字母大写。
命名规范：前缀为名称。
